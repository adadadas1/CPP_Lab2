## auto — автоматический вывод типа

`auto` инструктирует компилятор вывести тип переменной из инициализирующего выражения. Тип определяется **только при компиляции** — никакого runtime-overhead нет.

```cpp
int i = 10;
auto ai  = 10;      // int
auto ai2 = i;       // int
```

### Правила вывода auto

`auto` работает по тем же правилам, что и вывод шаблонного параметра:

- **константность** и **ссылочность** снимаются при копировании:
  ```cpp
  const int ci = 42;
  auto aci = ci;          // int (не const int!)

  const int& ref = ci;
  auto aref = ref;        // int (не const int&!)
  ```

- **volatile** тоже снимается при копировании.

- **Указатели** сохраняют свои квалификаторы:
  ```cpp
  const int* cptr = &i;
  auto acptr = cptr;      // const int*  (const у указуемого сохраняется)

  const int* const cptrc = &i;
  auto acptrc = cptrc;    // const int*  (верхний const снимается)
  ```

- **Строковые литералы**:
  ```cpp
  auto s = "hello";       // const char* (не std::string!)
  ```

### auto в возвращаемом типе функции

```cpp
auto add(int a, int b) { return a + b; }   // int
```

Если функция имеет несколько `return` с разными типами — ошибка компиляции.

## decltype — точный вывод типа

`decltype(expr)` выводит тип выражения **без снятия** константности, ссылочности, volatile:

```cpp
const int ci = 0;
decltype(ci) dci = 0;       // const int

const int& ref = ci;
decltype(ref) dref = ci;    // const int&

int* ptr = nullptr;
decltype(ptr) dptr = nullptr; // int*
```

### decltype(auto)

Комбинация: сохраняет точный тип как `decltype`, но с удобством `auto`:

```cpp
const int* ptr = &i;
decltype(auto) daptr = ptr;  // const int*
```

## Функторы

Функтор — класс с перегруженным `operator()`. Позволяет использовать объект как функцию:

```cpp
struct Multiplier {
    int factor;
    explicit Multiplier(int f) : factor(f) {}
    int operator()(int x) const { return x * factor; }
};

Multiplier times3(3);
times3(7);   // 21
```

Функторы хранят **состояние** (поля класса) — в отличие от обычных функций.

## Лямбда-функции

Лямбда — синтаксический сахар над анонимным функтором. Компилятор генерирует класс с `operator()` автоматически.

### Синтаксис

```
[захват](параметры) -> тип_возврата { тело }
```

- `тип_возврата` можно опустить — выводится автоматически.

```cpp
auto add = [](int a, int b) { return a + b; };
add(2, 3);  // 5
```

### Захват переменных (capture)

| Синтаксис | Что захватывается |
|-----------|-------------------|
| `[]` | ничего |
| `[=]` | все локальные переменные по значению |
| `[&]` | все локальные переменные по ссылке |
| `[x]` | переменная `x` по значению |
| `[&x]` | переменная `x` по ссылке |
| `[x = expr]` | захват с инициализацией (C++14) |

```cpp
int threshold = 10;
auto isAbove = [threshold](int v) { return v > threshold; };

int sum = 0;
auto accumulate = [&sum](int v) { sum += v; };
```

### auto в параметрах лямбды (C++14)

```cpp
auto add = [](auto a, auto b) { return a + b; };
add(1.5, 2.7);   // 4.2 (double)
add(1, 2);       // 3 (int)
```

### Лямбда vs. функтор

```cpp
// Эквивалентны:
struct Closure {
    int z;
    explicit Closure(int z_) : z(z_) {}
    int operator()(int a, int b) const { return z + a + b; }
};

auto lambda = [z = 100](int a, int b) { return z + a + b; };
```

### Лямбды в алгоритмах STL

Лямбды особенно полезны там, где раньше писали функторы или функции:

```cpp
std::vector<IP> result;
std::copy_if(ips.begin(), ips.end(), std::back_inserter(result),
    [b1](const IP& ip) { return std::get<0>(ip) == b1; });

std::sort(v.begin(), v.end(),
    [](const IP& a, const IP& b) { return a > b; });
```

## Кортежи (std::tuple)

`std::tuple` — обобщение `std::pair` на произвольное число элементов разных типов.

### Создание

```cpp
auto t = std::make_tuple(std::string("Alice"), 20, 3.14);
// тип: std::tuple<std::string, int, double>
```

### Доступ к элементам

По индексу:
```cpp
std::get<0>(t);   // "Alice"
std::get<1>(t);   // 20
```

По типу (если тип уникален в кортеже):
```cpp
std::get<std::string>(t);  // "Alice"
std::get<double>(t);       // 3.14
```

### Возврат нескольких значений

```cpp
auto divmod(int a, int b) {
    return std::make_tuple(a / b, a % b);
}

auto [quotient, remainder] = divmod(17, 5);  // C++17 structured bindings
```

### IP-адрес как кортеж

В данной лабораторной работе `IP = std::tuple<int, int, int, int>` — это позволяет:
- хранить 4 октета типизированно;
- использовать лексикографическое сравнение кортежей (`operator>`) для сортировки в численном порядке вместо строкового;
- обращаться к отдельным октетам через `std::get<N>`.

```cpp
using IP = std::tuple<int, int, int, int>;

// Обратная лексикографическая сортировка (численная, не строковая)
std::sort(ips.begin(), ips.end(), [](const IP& a, const IP& b) {
    return a > b;
});
// 1.10.1.1 > 1.2.1.1, т.к. 10 > 2
```

## decltype в контексте задачи

```cpp
decltype(ip_pool)::value_type ip = parse_ip(v.at(0));
// эквивалентно: IP ip = parse_ip(v.at(0));
// но не зависит от конкретного типа ip_pool
```

Это демонстрирует практическое применение `decltype` для обобщённого кода.

## Краткая сводка для защиты

### Ключевые определения

- **`auto`** — ключевое слово C++11; компилятор выводит тип из инициализирующего выражения. Тип фиксируется на этапе компиляции.
- **`decltype(expr)`** — получить тип выражения без его вычисления. `decltype(ip_pool)::value_type` — тип элемента вектора.
- **Лямбда-функция** — анонимная функция вида `[capture](params){ body }`. Создаёт объект с `operator()`.
- **Захват (capture)**: `[=]` — по значению, `[&]` — по ссылке, `[x]` — конкретная переменная по значению.
- **`std::tuple`** — гетерогенный набор значений фиксированного размера. Доступ: `std::get<N>(t)`.
- **`std::copy_if`** — алгоритм STL; копирует элементы, удовлетворяющие предикату. Принимает лямбду.

### Как работает программа

1. Читает строки из `stdin` вида `ip\ttext2\ttext3\n`
2. Разбивает по `\t` (`split`), берёт первое поле — IP-адрес
3. `parse_ip` разбивает IP по `.` → `std::tuple<int,int,int,int>`
4. Сортирует пул в обратном лексикографическом порядке (сравнение кортежей — побайтово)
5. Фильтрует и выводит 3 подсписка: `first==1`, `first==46 && second==70`, любой байт `==46`

### Ключевые вопросы на защите

**Q: Почему для IP используется `std::tuple`, а не `struct`?**  
A: Кортеж автоматически поддерживает лексикографическое сравнение (`<`, `>`) через `std::get`, что позволяет сортировать IP без написания компаратора вручную.

**Q: Что произойдёт, если написать `auto x = {1,2,3}`?**  
A: `x` будет иметь тип `std::initializer_list<int>`, а не `std::vector`. Это частый сюрприз с `auto`.

**Q: Чем `[b1]` в лямбде отличается от `[&b1]`?**  
A: `[b1]` — захват по значению (копия), `[&b1]` — по ссылке. При захвате по ссылке важно следить за временем жизни переменной.

**Q: Как работает `decltype(ip_pool)::value_type`?**  
A: `decltype(ip_pool)` — тип `ip_pool`, т.е. `std::vector<IP>`. `::value_type` — тип элемента, т.е. `IP`. Это позволяет не повторять тип явно.

**Q: Почему сортировка числовая, а не строковая?**  
A: IP хранится как `tuple<int,int,int,int>`. Сравнение кортежей — поэлементное по числовым значениям. `1.10.0.0 > 1.2.0.0` потому что `10 > 2`.

**Q: Что такое `std::back_inserter`?**  
A: Итератор-адаптор, который при разыменовании вызывает `push_back`. Используется в алгоритмах STL как вывод.
